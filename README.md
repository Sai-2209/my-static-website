# 🚀 Static Website Deployment using GitHub Pages

This project demonstrates how to build and deploy a simple static website using HTML and CSS through GitHub Pages. The objective of this task is to understand static hosting, version control using Git, repository management, and live website deployment using GitHub’s free hosting service.

The website was developed using HTML5 and CSS3, pushed to a GitHub repository, and deployed by enabling GitHub Pages from the repository settings (main branch, root folder). After configuration, GitHub automatically generated a live URL to access the website publicly.

Project Structure:
- index.html
- style.css
- README.md

Deployment Process:
1. Created index.html and style.css files.
2. Initialized Git repository and committed files.
3. Pushed project to a new GitHub repository.
4. Enabled GitHub Pages from Settings → Pages.
5. Selected main branch and root folder.
6. Accessed the live website link generated by GitHub.


Learning Outcome:
Through this task, I learned how static websites are hosted, how Git version control works, how to manage repositories, and how to deploy web content for free using GitHub Pages.
